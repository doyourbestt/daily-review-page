# 元气复盘局 - 管理员指南

## 📋 项目概述

这是一个**纯静态的复盘分享页面**，专为移动端（Android + 微信浏览器）优化。用户可以浏览每天的复盘内容、查看历史记录、点赞评论等互动功能。

### ✨ 核心特性

- ✅ **纯静态架构** - 无需服务器，直接部署HTML文件
- 📅 **历史存档** - 支持多天内容存储和切换
- ❤️ **互动功能** - 点赞、评论、收藏（LocalStorage本地存储）
- 📱 **移动优化** - 动态视口高度、安全区域适配、低端设备检测
- 🎥 **智能视频** - 自动压缩、渐进加载、降级方案
- 🚀 **高性能** - 骨架屏、懒加载、动画优化

## 🔧 如何添加新一天的复盘内容

### 方法：编辑 `data/reviews.json` 文件

#### 步骤 1: 打开文件

使用任意文本编辑器打开 `data/reviews.json`（推荐：VS Code、Notepad++、记事本）

#### 步骤 2: 复制以下模板

```json
"2026-02-12": {
  "title": "元气复盘局｜新的一天",
  "subtitle": "每一天都在发光发热 ✨",
  "intro": {
    "heading": "🌟 今日主题标题",
    "content": "今日的介绍文字，可以是欢迎语或主题说明～"
  },
  "people": [
    {
      "name": "成员名字",
      "emoji": "🐱",
      "role": "成员角色/标签",
      "content": [
        "第一条分享内容",
        "第二条分享内容",
        "第三条分享内容（可以添加更多）"
      ]
    },
    {
      "name": "另一位成员",
      "emoji": "🧠",
      "role": "另一个角色",
      "content": [
        "这位成员的分享内容1",
        "分享内容2"
      ]
    }
  ],
  "summary": "今日的总结文字，可以是对整体复盘的概括和展望～"
}
```

#### 步骤 3: 粘贴到正确位置

在 `reviews.json` 文件中，找到最后一个日期条目（例如 `"2026-02-11"`），在它**之前**添加新内容：

```json
{
  "2026-02-12": {
    "title": "...",
    ...
  },
  "2026-02-11": {
    "title": "...",
    ...
  },
  "2026-02-10": {
    ...
  }
}
```

⚠️ **注意逗号**：确保每个日期条目之间用**英文逗号**分隔！

#### 步骤 4: 保存文件

保存 `reviews.json` 文件后，刷新网页即可看到新内容。

## 📝 内容填写注意事项

### ✅ 必填字段

- **日期** (格式: `YYYY-MM-DD`)
- **title** - 页面大标题
- **subtitle** - 副标题
- **intro.heading** - 介绍部分标题
- **intro.content** - 介绍部分正文
- **people** - 至少1位成员
  - **name** - 成员名字
  - **emoji** - 成员表情（推荐使用单个emoji）
  - **role** - 成员角色标签
  - **content** - 分享内容列表（至少1条）
- **summary** - 总结文字

### 🎨 Emoji 推荐

可以在 [Emojipedia](https://emojipedia.org/) 搜索合适的表情：
- 人物: 🧑‍💻 👩‍🎨 🧙‍♂️ 🦸‍♀️
- 动物: 🐱 🐶 🦊 🐻 🦁
- 物品: 💻 📚 🎨 ✍️ 🎸
- 自然: ☀️ 🌟 🌈 🌸

### ⚠️ 特殊字符处理

- **引号**：使用 `\"` 转义（例如：`"她说\"你好\""`）
- **换行**：不支持，请拆分为多条内容
- **emoji**：可以直接使用，无需转义

## 🗑️ 如何删除旧内容

如果某天的内容不想再显示，直接删除对应的整个日期条目即可：

```json
{
  "2026-02-12": {...},
  // 删除 "2026-02-11" 整个块
  "2026-02-10": {...}
}
```

⚠️ 注意：删除后记得调整**逗号**，避免JSON格式错误。

## ❓ 常见问题

### Q1: 修改了 `reviews.json` 但页面没更新？

**A:** 清除浏览器缓存并强制刷新（手机端：清除微信缓存或重新分享链接）

### Q2: 视频不播放怎么办？

**A:** 这是正常的降级行为。低端设备或网络不佳时会自动显示静态背景图。如果想强制显示视频，可以：
1. 检查 `videos/hero_compressed.mp4` 文件是否存在
2. 检查视频文件是否损坏（可以在电脑上用播放器测试）
3. 确保视频路径正确：`./videos/hero_compressed.mp4`

### Q3: 点赞/评论数据丢失了？

**A:** 这些数据存储在浏览器的 LocalStorage 中。如果用户清除了浏览器数据或更换设备，数据会丢失。这是**纯静态架构**的限制，无需后端数据库。

### Q4: 如何更换背景视频？

**A:** 
1. 准备新视频（推荐格式：MP4，分辨率：1080p，时长：6-10秒循环）
2. 使用视频压缩工具压缩到 5MB 以下（推荐工具：HandBrake、FFmpeg）
3. 替换 `videos/hero_compressed.mp4` 文件
4. 更新 `images/hero_fallback.jpg` 为视频的代表帧（可选）

### Q5: 如何修改颜色主题？

**A:** 在 `index.html` 中找到 CSS 变量部分（约第30行），修改颜色值：

```css
:root {
    --color-primary: #FF6B35;   /* 主色调 */
    --color-secondary: #F7931E; /* 辅助色 */
    --color-accent: #FFD23F;    /* 强调色 */
}
```

### Q6: JSON 格式错误怎么办?

**A:** 使用在线 JSON 验证工具检查：
- [JSONLint](https://jsonlint.com/)
- [JSON Formatter](https://jsonformatter.curiousconcept.com/)

常见错误：
- 缺少逗号
- 多余的逗号（最后一项后面不能有逗号）
- 引号不匹配
- 中文引号（应使用英文引号 `"` 而非 `""`）

## 🛠️ 技术细节

### 性能优化

- **低端设备检测**：自动检测设备内存和CPU核心数，低于2GB或2核心时跳过视频和复杂动画
- **动态视口**：使用 `100dvh` 而非 `100vh`，避免移动端地址栏显示/隐藏导致的抖动
- **安全区域适配**：使用 `env(safe-area-inset-*)` 适配刘海屏和手势条
- **懒加载**：视频使用 Intersection Observer 懒加载
- **字体优化**：使用 `font-display: swap` 避免字体加载阻塞

### LocalStorage 数据结构

```javascript
// 点赞数据
localStorage.setItem('review_likes', JSON.stringify({
  "2026-02-11:喵喵": true,
  "2026-02-11:铭阳": false
}));

// 评论数据
localStorage.setItem('review_comments', JSON.stringify({
  "2026-02-11:喵喵": [
    {"name": "匿名", "text": "太棒了！", "time": "2026-02-11 10:30"}
  ]
}));

// 收藏数据
localStorage.setItem('review_favorites', JSON.stringify([
  "2026-02-11:喵喵",
  "2026-02-10:达达"
]));
```
